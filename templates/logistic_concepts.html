Basic Concepts

Understanding Logistic Regression applied to E-Commerce Savings Prediction.

What is Logistic Regression?

Logistic Regression is a supervised ML algorithm used for binary classification — it predicts whether something belongs to one of two categories (0 or 1). Unlike Linear Regression which outputs a continuous number, Logistic Regression outputs a probability between 0 and 1 using the Sigmoid function.

In this model: We predict whether an e-commerce purchase will generate High Savings (discount ≥ 60 Rs.) or Low Savings based on product price, category, payment method, purchase month, and whether it's a weekend.

The Sigmoid Function

Converts any number into a probability between 0 and 1:

σ(z) = 1 / (1 + e⁻ᶻ)
z = b₀ + b₁·Price + b₂·Category + b₃·Payment + b₄·Month + b₅·Weekend
σ(z) ≥ 0.5 → Predict High Savings (1)
σ(z) < 0.5 → Predict Low Savings (0)

How the Model Was Trained
1
Dataset
3,660 e-commerce transactions. Target: savings ≥ 60 Rs. → 1,095 High / 2,565 Low.
2
Feature Engineering
Extracted month and is_weekend from Purchase_Date. Label-encoded Category and Payment Method.
3
StandardScaler
All features normalized to mean=0, std=1 so no single feature dominates due to scale.
4
80/20 Train-Test Split
2,928 records for training, 732 for evaluation. Result: 77% accuracy — realistic and credible.
Evaluation Metrics

Accuracy — 77%

77 out of every 100 predictions are correct. Realistic — a 99% would suggest the model is cheating.

Precision

Of all purchases predicted as "High Savings", how many actually were? Penalizes false alarms.

Recall

Of all actual "High Savings" purchases, how many did the model detect? Penalizes missed cases.

F1-Score

Harmonic mean of Precision and Recall. Best single metric when classes are imbalanced.
